# Historical Shipped Ledgers (0.1.x – 0.3.0) — 天衡 (Tianheng)

This document contains historical records of shipped capabilities and architectural milestones from version `0.1.0` through `0.3.0`.

Per `AGENTS.md` and `PROJECT.md`, active capability requirements live in [`openspec/specs/*`](../../openspec/specs) and active contract decisions live in [`PROJECT.md`](../../PROJECT.md). This file is preserved for historical reference and to avoid cluttering [`BACKLOG.md`](../../BACKLOG.md).

---

## 0.3.0 Identity Migration Bundle (SHIPPED)

The `0.3.0` identity migration bundle landed in one coordinated OpenSpec change (`feat(xuanji)!: make reaction identity semantic`):
- **Unsafe-site fact decomposition**: owner, owner-kind, trait, and name as separate structured fields (`semantic-unsafe-confinement`).
- **Async seam-vs-signature decision**: identity is the seam (module, owner-kind, owner, name); the rendered signature stays presentation-only (`semantic-async-exposure-boundary`).
- **Stable rule identity**: rule key separated from human presentation (`RuleKey` vs. `rule` — `structured-violation-identity`).
- SARIF fingerprints and baseline wire moved with it; no numeric baseline is read, and no automatic migration off old baselines is executed.

## 0.2.x Deliberate Breaking & Truth Repair Window (SHIPPED)

- **Structured Violation Identity**: Moved live findings to dimension-owned `FindingKey`s and version-2 baselines.
- **`guibiao` Metadata Substrate**: Shared `cargo metadata` reader extracted into `xingbiao` beneath all observation dimensions.
- **Composed Adopter Surface**: Compile-reacted against adopter usage (`tianheng::prelude::*`).
- **Unconditional `#[path]` Following**: Modules with `#[path = "..."]` are followed across all 3 observation dimensions.
- **`resolve_self_type` Hop-Cap**: Resolved divergent rewrite loop vulnerability in `hunyi`.
